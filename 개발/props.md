1. Props의 기본 개념

React 컴포넌트는 하나의 독립된 함수처럼 동작합니다. 그리고 props는 그 함수가 외부에서 전달받는 매개변수(parameter) 역할을 합니다.

즉, 컴포넌트는 직접 데이터를 만들기보다 상위 컴포넌트로부터 props를 받아 화면을 구성합니다.
→ “부모 컴포넌트가 자식 컴포넌트에게 주는 데이터”

2. Props가 중요한 이유
   (1) 컴포넌트 재사용성 ↑

같은 컴포넌트를 다양한 값으로 사용 가능하도록 만들어 생산성을 크게 높입니다.
→ 같은 버튼 컴포넌트라도 텍스트만 props로 바꾸면 여러 곳에 사용 가능

(2) 데이터 흐름을 명확하게 유지

React는 기본적으로 단방향 데이터 흐름(one-way data flow) 을 사용합니다.
데이터가 항상 “위에서 아래로” 흐르기 때문에 구조가 단순하고 유지보수성이 좋아집니다.

(3) 상태(state)와의 역할 분리

state: 컴포넌트 내부에서 관리하는 변화하는 값

props: 외부에서 전달받는 고정된 값(자식은 수정 불가)

이 구분 덕분에 UI의 동작을 예측하기 쉬워집니다.

3. Props의 핵심 특징

읽기 전용(Read-only)
자식 컴포넌트는 props를 직접 변경할 수 없습니다.
(데이터를 변경해야 한다면 반드시 부모가 상태를 수정하고 다시 내려줍니다.)

객체 형태로 전달됨
컴포넌트 내부에서는 props.title, props.onClick처럼 접근합니다.

함수도 전달 가능
단순 텍스트나 숫자뿐 아니라 함수도 props로 전달할 수 있어 이벤트 처리에 유용합니다.
(ex. 버튼 클릭 시 부모의 함수 실행)
