1. State의 기본 개념

State는 컴포넌트 내부에서 관리되는 변하는 데이터입니다.
UI는 이 state 값에 따라 화면을 다시 렌더링(render)하므로, 상태는 React가 동적인 화면을 구성하는 핵심 요소라고 할 수 있습니다.
(한국어 요약: 컴포넌트 안에서 바뀌는 값)

2. State가 필요한 이유
(1) 사용자 인터랙션 대응

입력 값, 버튼 클릭, 모달 열림/닫힘 등 사용자 행동에 따라 화면이 달라져야 합니다.
이 변화의 기준이 되는 값이 바로 state입니다.

(2) 화면 렌더링 자동화

State가 변경되면 React는 해당 컴포넌트를 자동으로 다시 그립니다.
개발자가 DOM을 직접 업데이트할 필요가 없으므로 유지보수성이 크게 향상됩니다.

(3) 컴포넌트의 독립성 강화

각 컴포넌트는 자신의 state를 보유함으로써 독립적으로 동작할 수 있습니다.
이 구조는 규모가 커질수록 유리합니다.

3. State의 특징
(1) 내부에서만 관리됨 (Internal only)

Props는 외부(부모)에서 전달되지만,
State는 컴포넌트가 스스로 생성하고 스스로 수정합니다.

(2) 변경 시 자동 렌더링

UI는 state 값과 완전히 동기화됩니다.
→ 값이 바뀌면 화면도 자동으로 바뀜

(3) 직접 변경 불가, 반드시 setter 사용

React는 state를 직접 수정하면 렌더링을 하지 않기 때문에,
항상 setState 혹은 useState의 setter 함수를 사용해야 합니다.

4. Props와 State의 차이 (핵심 요약)
구분	Props	State
데이터 출처	부모 컴포넌트	해당 컴포넌트 내부
수정 가능 여부	자식은 수정 불가	직접 수정 가능(Setter 사용)
역할	외부에서 전달되는 정보	UI 변화의 중심 데이터
렌더링	값이 바뀌면 렌더링	값이 바뀌면 렌더링
5. 짧은 한국어 핵심 번역

State = 컴포넌트 내부에서 바뀌는 값

UI 변화의 핵심 원인

Setter로만 수정 가능합니다
